06

Panels & components

GitHub panel

  • Header: repo scope + open-PR count
  • Scrollable PR rows: repo, #num, age, title
  • Status row: checks (pass/failing), review state, rebase/draft flags
  • Color-codes failing (pink) vs passing (teal)
  • Views: open-prs · review-requests · notifications
shadcn · Card · ScrollArea · Badge · Separator

Flue chat harness

  • Header: agent file + durable ctx size
  • User bubbles (right, cyan) · agent replies (left, teal rule)
  • Tool-call chips: workflow · step n/m · call()
  • Scrollback persists across restarts (durable)
  • Bound to any Flue agent via layout[].agent
shadcn · Card · ScrollArea · Badge · Avatar

Composer

  • Pinned bottom of chat column, ~44px
  • Prompt glyph (pink) + input + blinking cursor
  • Hint affordances: ⏎ send · ⌃K tools
  • Submits to the bound agent's run loop
shadcn · Textarea · Button · kbd
07

Behaviors & states

ALWAYS-ON

Renders full-screen, no chrome, no scrollbars. Survives restarts; state is restored from the durable harness on boot.

POLLING

GitHub polls on a 60s interval (or webhook). Sensors at 1s, flue ping at 2s. All cadences configurable per panel.

EMPTY / ERROR

No PRs → muted "inbox zero" row. Offline harness → flue:offline in status, composer disabled, last scrollback retained.

RESPONSIVE

Layout is the 32:9 target but reflows to any resolution from config. Below 12 effective cols, panels stack by priority order.

Wire the tokens above into shadcn's CSS variables (--primary = teal, --destructive = pink, --ring = cyan) so the generated components inherit the Miami-camo theme. Panels are plain Cards on a grid-cols-12; only the status line and tiger-stripe theming need custom CSS. Dark and light ship from the same tokens — flip via data-theme (see the toggle in §01) using the deck.light* values.
