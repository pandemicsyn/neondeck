An always-on agent for the way you ship.

Always watching your repos

The GitHub sidebar never sleeps. Open PRs, failing checks and review requests stay in the corner of your eye while you work — no tab to refresh.

Durable Flue harness

The agent runs on Flue's programmable TypeScript runtime. Workflows are durable — they survive restarts and reboots.

A second monitor that thinks

Tuned for the 2560×720 Xeneon Edge, but the grid is fully configurable — pin it to a spare monitor, a laptop sidebar, anything always in view.

Local-first & private

Runs on your dev machine. Your tokens, your code, your model — bring your own provider. No account, no cloud, no telemetry.

Miami tiger-stripe theming

Ships with the Miami-camo preset. Every color, stripe and glow is a theme token — make it yours.

Hackable in TypeScript

Panels, tools and sensors are plain TS modules. If you can write a function, you can extend the deck.

// ONE FILE

The whole deck is a config file.

Declare your display, theme, panel spans and status-line sensors. Point the chat panel at any Flue agent. Reload and it's live.

~/.config/neondeck/neondeck.config.ts

// neondeck.config.ts
import { defineDeck } from "neondeck";

export default defineDeck({
  display: { width: 2560, height: 720 },
  theme: "miami-tiger",
  layout: [
    { panel: "github", span: 4, props: { view: "open-prs" } },
    { panel: "flue-chat", span: 8, agent: "./agents/triage.ts" },
  ],
  statusline: ["cpu", "mem", "gpu", "net", "flue.status", "clock"],
});
